# 纵横四海（zhsh-game）项目大纲

## 一、项目概述

**类型**：Seafaring RPG 文字游戏（网页端 + 管理后台）  
**技术栈**：Node.js(Express) + MySQL + Vue3 + WebSocket  
**定位**：经典航海冒险文字mud游戏的现代化复刻版

## 五、开发阶段规划

### 阶段一：核心循环跑通（优先级最高）

目标是让玩家能完成"注册→打怪→升级→换装→完成任务"的核心循环。

**1.1 怪物扩充**（紧急）
- 当前37只怪物太少，每个地图区域需要10-20只
- 按海域/地图分配怪物等级（1-60级区间）
- 配置掉落表（monster_drop），当前90条太少
- 来源数据：参考老曾PHP版怪物数据

**1.2 NPC + 商店完善**（紧急）
- 每个城市至少6个NPC（码头/广场/市场/酒馆/铁匠铺/商店）
- NPC商店物品上架（npc_shop_item）
- NPC对话框内容（npc_dialog）
- 5个现有NPC扩展到100+个

**1.3 任务扩充**（紧急）
- 每个城市5-10个任务
- 主线任务：跑商/航海/打怪/送信
- 支线任务：地图探索/NPC故事
- 奖励：钱/经验/物品

**1.4 客户端接口对接**（紧急）
- 银行 BankView → 对接 /api/bank 路由
- 赌场 CasinoView → 对接 /api/casino 路由
- 市场 MarketView → 对接 /api/market 路由
- 聊天 ChatView → 对接 /api/chat 路由
- 好友 FriendView → 对接 /api/friend 路由

**1.5 航海系统**（高优先级）
- 跨海域航行：从当前地图移动到相邻海域
- 航海界面 SailView 已有，需对接后端
- 航海消耗：时间/食物/船只耐久
- 遇怪：海上航行随机遇怪

### 阶段二：经济系统完善

**2.1 货物贸易系统**
- 港口商品价格波动（market_price）
- 低买高卖：不同港口同一商品价格不同
- 货物种类：丝绸/香料/矿石/香料/木材等
- 贸易任务：运送指定货物到某港口

**2.2 银行系统**
- 存取钱功能（防止死亡掉落）
- 利息机制
- 银行日志

**2.3 打造/强化系统**（smith.js）
- 铁匠铺打造装备
- 强化/升级装备
- 材料消耗

### 阶段三：社交与成长

**3.1 公会系统**（guild.js）
- 创建/加入公会
- 公会成员管理
- 公会建筑/科技

**3.2 船员系统**
- 招募船员（不同属性/技能）
- 船员装备
- 船员技能影响航海/战斗

**3.3 宠物系统扩充**
- 宠物抓捕（使用抓宠道具）
- 宠物进化
- 宠物参战

**3.4 礼包/CDKEY系统**
- 激活码兑换：礼包ID + 兑换码
- 一次性码/限次码
- 后台生成礼包

**3.5 排行榜**
- 等级榜/财富榜/战斗力榜
- 爵位/航海排名

### 阶段四：内容扩充

**4.1 地图地下城**
- 地下宫殿（副本）
- 轩辕帝陵
- 各城市副本

**4.2 爵位/称号系统**
- 航海爵位（从水手到提督）
- 战斗称号
- 收集称号

**4.3 剧情系统**
- 主线剧情推进
- NPC背景故事
- 结局分支
